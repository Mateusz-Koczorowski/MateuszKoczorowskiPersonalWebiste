# Story 001: Repository Foundation

## Goal

Set up a production-ready foundation for the Senior QA Engineer personal website.

## Scope

- Next.js App Router
- TypeScript strict mode
- Tailwind CSS v4
- shadcn/ui-compatible structure
- Framer Motion foundation
- ESLint and Prettier
- Documentation starter set

## Acceptance Criteria

- Project directories exist: `app`, `components`, `content`, `docs`, `docs/stories`, `public`, and `tests`.
- The home page uses semantic HTML.
- Metadata, robots, and sitemap foundations exist.
- Documentation includes PRD, branding, UX design, technical architecture, AI guidelines, backlog, and implementation stories.
- No lorem ipsum, fake metrics, or confidential project names are introduced.
- Accessibility criteria from `docs/09-accessibility.md` are considered for the scaffold.
- Responsive criteria from `docs/10-responsive-design.md` are considered for the scaffold.
