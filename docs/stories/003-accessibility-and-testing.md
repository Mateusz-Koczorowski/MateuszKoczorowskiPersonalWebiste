# Story 003: Accessibility and Testing Foundation

## Goal

Add automated checks that protect accessibility, build health, and core rendering behavior.

## Acceptance Criteria

- Add a test runner decision record.
- Add accessibility smoke tests for the home page.
- Add CI-ready commands for linting, typechecking, formatting, and production build.
- Document manual QA checks for keyboard navigation, reduced motion, and responsive behavior.
- Include WCAG 2.2 AA-oriented checks where practical.
- Include future guidance for Playwright, axe-core, Lighthouse, or similar tools without adding unnecessary tooling early.
- Confirm accessibility criteria from `docs/09-accessibility.md` and responsive criteria from `docs/10-responsive-design.md` are represented in the test plan.

Planned.
