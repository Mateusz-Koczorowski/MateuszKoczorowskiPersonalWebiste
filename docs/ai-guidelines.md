# AI Guidelines

## Purpose

AI may support QA research, drafting, test design, refactoring ideas, and content iteration. Human review remains mandatory.

## Allowed Uses

- Draft exploratory testing charters
- Generate candidate test cases from requirements
- Summarize public technical notes
- Suggest automation maintenance improvements
- Help structure sanitized case studies
- Improve accessibility and SEO wording

## Required Review

Every AI-assisted output must be checked for:

- Accuracy
- Security impact
- Confidentiality risk
- Unsupported claims
- Bias or misleading framing
- Maintainability

## Prohibited Uses

- Publishing confidential project details
- Inventing metrics, client names, or outcomes
- Treating generated tests as complete coverage
- Shipping AI-authored technical claims without verification

## Disclosure Principle

When content materially depends on AI-assisted work, keep internal notes about how it was reviewed. Public disclosure can be added where appropriate for articles or talks.
