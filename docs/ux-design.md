# UX Design

## Experience Goals

- Make the role and expertise clear in the first viewport.
- Help visitors scan focus areas quickly.
- Preserve trust through concise, specific language.
- Keep navigation simple until the content library grows.
- Prioritize readability, keyboard access, and reduced-motion support.

## Information Architecture

- Home
- Focus areas
- Case studies
- Speaking and community
- Writing
- Contact

Only Home and the foundational focus area section are implemented in the initial scaffold.

## Accessibility Requirements

- Semantic landmarks for header, navigation, main content, and sections
- Keyboard-accessible navigation
- Visible focus states
- Sufficient color contrast
- Accessible labels for icon-only controls
- Correct native elements for links, buttons, and other interactions
- Reduced-motion support for animations
- Descriptive headings and link text

## Responsive Behavior

Design mobile-first. The first release should work cleanly on narrow screens, tablets, laptops, and wide desktop viewports without relying on hover-only interactions.

Use the responsive rules in `docs/10-responsive-design.md` for breakpoints, layout behavior, and acceptance criteria.
